ARCHIVO: build.gradle
TIPO: Gradle build script (Groovy)
ROL: Configuracion global de plugins para el proyecto raiz.
DETALLE (PASO A PASO):
1) buildscript { repositories { google(), mavenCentral() } } define repositorios para plugins legacy.
2) plugins { id("com.android.application") version "8.2.2" apply false } fija version de AGP.
3) plugins { id("org.jetbrains.kotlin.android") version "1.9.10" apply false } fija version de Kotlin.
4) apply false indica que los plugins se aplican en modulos, no en el root.
DEPENDENCIAS / VERSIONES:
- com.android.application: 8.2.2
- org.jetbrains.kotlin.android: 1.9.10
EFECTO:
- Centraliza versiones de plugin para todos los modulos del proyecto.

ARCHIVO: gradlew.bat
TIPO: Script Windows (batch)
ROL: Lanza el Gradle Wrapper en entornos Windows.
DETALLE:

ARCHIVO: app/src/main/java/com/lapmaster/MainActivity.kt
TIPO: Kotlin (Activity + Compose)
ROL: Activity principal y navegacion por pantallas.
DETALLE (PASO A PASO):
1) data class ElementoNavegacion: encapsula Pantalla, etiqueta e icono.
2) class ActividadPrincipal:
   - Instancia ModeloVistaLapMaster via viewModels.
   - Gestiona ProveedorGpsNativo y sensores de rotacion.
   - Define permisos de ubicacion y ActivityResult para solicitarlos.
3) onCreate:
   - Ajusta WindowInsets y status bar.
   - setContent { AplicacionLapMaster(modeloVista) }.
   - Inicializa proveedor GPS y sensor brujula.
4) onStart: solicita permiso GPS si es necesario.
5) onResume/onPause: registra o desregistra listener del sensor.
6) inicializarProveedorGps:
   - Crea ProveedorGpsNativo con callback a modeloVista.gps.alActualizarEstadoGps.
7) solicitarPermisoGpsSiEsNecesario:
   - Si hay permiso, inicia GPS; si no, lanza request.
8) Sensor:
   - listenerBrjula procesa Sensor.TYPE_ROTATION_VECTOR.
   - Obtiene matriz de rotacion, ajusta por orientacion de pantalla.
   - Calcula azimut y actualiza rumbo en el ViewModel.
9) ajustarMatrizPorRotacionPantalla: remapea ejes segun Surface.ROTATION_90/180/270.
10) obtenerRotacionPantalla: API >= R usa display.rotation; si no, windowManager.
11) @Composable AplicacionLapMaster:
   - Observa estadoUi con collectAsState.
   - Determina accionActiva (cronometros o sectores activos).
   - Define lista de pantallas y pager state.
   - LaunchedEffect: sincroniza pagina con estado de navegacion.
   - Oculta barra superior si horizontal + en tiempos + cronometro activo.
   - PantallaConAccionActiva(activo) mantiene pantalla encendida.
   - TemaLapMaster aplica tema claro/oscuro.
   - Scaffold + HorizontalPager renderiza PantallaMenu, PantallaClima, PantallaVueltas, PantallaSectores, PantallaGraficas.
12) @Composable BarraMenuSuperior:
